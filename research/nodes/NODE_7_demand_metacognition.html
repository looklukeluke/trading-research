需求侧/超大规模节点 — 24轮元思考过程

从"买家确认需求"到"LOI不是合同，CEO承诺不是CapEx"  |  2026-03-21
研究笔记 — 思考过程比结论更有价值

24轮元思考 9个Phase 3+4代理人交叉验证 6条投资信号

🧠 24轮元思考的核心结论

  1. 需求侧的真正问题不是"需求是否真实"（95%确认），而是"2028年后是否可持续" — $250B CapEx vs $50-80B AI应用收入 = 3-5x gap。历史先例：1998-2001光纤泡沫有相同的CapEx/收入比
  2. 三位代理人都是买家，但他们有动机表现承诺 — CEO公开宣布CapEx后再缩减是"职业自杀"。Altman的$100B LOI、Nadella的$80B指引、Musk的Terafab公告都有不可逆的叙事锁定效应
  3. 最被低估的信号：Altman的Broadcom芯片延期（Q1→Q3 2026） — 自研芯片比预期更难 = NVIDIA依赖性短期反而加强。但长期方向不变：所有买家都在建"逃生通道"
  4. 中国需求侧（#24 DeepSeek, #25 ByteDance, #26 Alibaba, #28 Tencent）是供应链"隐形鲸鱼" — 他们的GPU采购规模与西方hyperscaler可比，但出口管制迫使他们走自研路线，加速了全球替代生态的发育
  5. Node 7页面最大的缺陷：只看了三个美国买家 — 全球AI需求有40%来自中国、中东和东南亚。这些需求流不经过Altman/Nadella/Musk但同样驱动NVIDIA收入
PHASE I — EXPLORE

任务定义：表面任务 vs 真正任务

#1 表面任务 vs 真正任务

表面任务：从买方视角验证半导体供应链的需求端是否真实。

真正任务：帮交易者回答 "$1T AI基建叙事到底有多少是真金白银、多少是LOI吹泡？如果泡沫破裂，最早的信号从哪里来？"

Nodes 1-6的16位代理人（Jensen、Lisa Su、Morris Chang等）全部是卖家。他们说"需求无限"是在自卖自夸。本节点的核心价值是买家验证：当花钱的人也说"需要更多"，这比卖家说的可信10倍。但买家也有动机——他们公开承诺后不能反悔。

#2 代理人选择的认知弧线

初始本能：选最大的三个GPU买家——Microsoft、Google、Amazon。标准hyperscaler三角验证。

反思后调整：Google/Amazon是买家兼卖家（TPU、Trainium），他们的"买更多GPU"叙事被自家产品的竞争立场污染。更纯粹的"买家"是：

  • Musk/xAI：100%买NVIDIA，零自研芯片。最纯的GPU买方信号
  • Altman/OpenAI：最大LOI + 正在做Broadcom自研 = 买方转型窗口最佳观测点
  • Nadella/Microsoft：最透明的CapEx数据（上市公司季报审计） + 49% OpenAI = 两个需求流汇聚点

遗憾：没有直接覆盖中国需求侧（DeepSeek、字节跳动、阿里、腾讯）。他们的GPU消费量可比西方hyperscaler，但出口管制创造了一个完全不同的需求动态。

#3 "就是这个" vs "不对"

成功标准：第一眼看到"需求真实度概率"+"LOI转化率评估"+"最早的泡沫信号在哪里"。

失败标准：三位CEO的公开言论堆砌（公关稿翻译）；"需求是真的"但不给概率和条件（和事佬）；忽略中国需求（美国中心偏差）。

PHASE II — EXPLORE

数据解码：三位买家的隐含叙事

#4 三位买家到底讲了什么故事？

隐藏叙事解码

核心发现：买家承诺偏差(Commitment Bias)。三位CEO公开宣布了巨额CapEx后，缩减 = 承认战略失误 = 股价崩盘 = 职业生涯终结。他们被自己的公告锁住了。Ben Thompson说得对："CapEx commitments are sunk cost once announced — pulling back publicly is career suicide for a CEO."

跨节点交叉信号

#5 中国需求侧的盲区

Node 7只覆盖了三个美国买家。但全球AI计算需求的分布是：

  • 美国hyperscaler: ~55-60%（MSFT/GOOG/AMZN/META + OpenAI/xAI）
  • 中国科技巨头: ~25-30%（ByteDance/Alibaba/Tencent + DeepSeek/Baidu）
  • 其他: ~10-15%（中东主权基金、日韩、欧洲）

中国需求侧的独特动态：出口管制迫使H20替代H100，但总购买量不减。DeepSeek用"穷人方法"（更少GPU、更高效架构）取得突破 → 这同时证明"不需要那么多NVIDIA GPU也能做前沿AI"和"对计算的需求仍然无限"。这两个信号互相矛盾，Node 7没有面对。

PHASE III — DIAGNOSE

矛盾与张力：Node 7没面对的四个矛盾

#6 四个Node 7没面对的矛盾
矛盾1：买家说"需求无限"但他们不是中立证人
Node 7的核心前提是"买家确认 = 需求真实"。但Altman说"需要更多GPU"时，他在向投资者证明$300B估值合理；Nadella说"供给受限"时，他在为$80B CapEx辩护。买家和卖家的动机方向一致：都需要"需求无限"叙事成立。真正的独立验证应来自最终用户（企业AI采购决策者），不是中间商。
矛盾2：$100B LOI被当作$100B CapEx引用
Altman的LOI是意向书，不是采购合同。Node 1的Jensen评分LOI可信度60%。但Node 7在三角验证表中把它和Nadella的审计CapEx并列，暗示同等硬度。LOI→合同转化率可能只有40-60%。$100B的60% = $60B — 这仍然巨大，但叙事冲击力完全不同。
矛盾3：三个代理人的时间线可信度差1000x
Nadella的$80B来自上市公司季报（审计师签字），可信度90%+。Altman的$100B LOI来自媒体报道，可信度60%。Musk的Terafab来自Twitter公告，可信度30%（历史x2-3延迟）。但三角验证表给了三人类似的权重。
矛盾4：所有人同意"需求无限" = 拥挤多头
卖家（Jensen）、买家（Altman/Nadella/Musk）、分析师（MS/GS）、投资者全部在同一侧。没有人公开做空AI CapEx。历史规律：所有人都同意的方向最后总会反转，争论的只是时间。Node 7缺少"Devil's Advocate"视角。
#7 信息空白
  1. $100B LOI中已签约合同比例（没人公开过）
  2. Copilot日活跃使用率 vs 购买座位数（5M seats可能只有500K daily active）
  3. xAI Memphis集群实际利用率（100K GPU中训练效率如何？idle比例？）
  4. 中国hyperscaler的实际GPU消费量（出口管制后的灰色渠道规模）
  5. AI应用端真正的企业采购意愿 vs CEO推动的"自上而下部署"

结论：需求侧信息也系统性偏向乐观。买家和卖家都在讲"AI是不可阻挡的大趋势"——因为他们的估值/股价/职业都依赖这个叙事。

PHASE IV — DIAGNOSE

时间与反面：加速 vs 减速

#8 加速 vs 减速
#9 最强反面论证：CapEx悬崖场景
"The hyperscaler AI race is the largest infrastructure buildout since the transcontinental railroad. The question is not whether it will be built — it's whether the ROI will materialize before the capital is exhausted." — Ben Thompson, Stratechery
  1. $250B CapEx vs $50-80B AI应用收入：3-5x gap。历史上这个比例只在1998-2001光纤泡沫期出现过
  2. ROI gap的时间窗口：如果2027年底AI应用收入仍不能达到$200B+，董事会压力将迫使CapEx缩减
  3. Copilot是关键验证点：如果企业AI最大的部署（5M seats）不能证明30-40%生产力提升是真实的、可持续的，整个"AI创造价值"叙事崩塌
  4. CEO承诺锁定的反面：当反转终于来临，因为CEO们已经把自己绑在了"all-in AI"上，缩减速度会比正常周期更快更猛（恐慌性缩减 vs 有序调整）
  5. 概率：15-20%。但如果发生，半导体全链暴跌50-70%，不是-10%的调整
#10 Altman $100B LOI vs Nadella Project Maia vs Musk Terafab：自研竞赛的真实进度

结论：自研芯片是真实趋势但进度远慢于叙事。所有项目都需3-5年才能有意义地减少NVIDIA依赖。这给了NVIDIA一个"confirmed runway"到2028-2029。之后？不确定性急剧上升。

PHASE V — BUILD

结构诊断：当前Node 7的优劣

#11 当前Node 7结构的优劣
#12 买家视角 vs 卖家视角的系统性盲区

Node 1（卖家视角）的盲区是"夸大自己的护城河"。Node 7（买家视角）的盲区是"夸大自己的需求合理性"。

两个节点合在一起的盲区：缺少"最终消费者"视角。GPU从NVIDIA卖给Hyperscaler，Hyperscaler卖AI服务给企业。企业是否愿意付足够的钱用AI来justify整条链？这个问题Altman/Nadella/Musk都回避了，因为答案还不清楚。

Copilot是最好的观测窗口。如果5M seats中daily active < 1M，那"AI创造价值"叙事的根基就很脆弱。

PHASE VI — VERIFY

成品审视：诚实的自我诊断

#13 可信度评分的口径问题

Musk被低估了。他的$25B GPU花费是可审计的，比Altman的LOI更"硬"。问题不是他的花销可信度（很高），而是他的未来承诺可信度（很低）。报告把两者混在一起了。

#14 Padding识别
  • Musk xAI估值$80B — 与需求侧分析关系弱（估值不是需求信号）
  • OpenAI非营利→营利重组 — 公司治理变化，不是GPU需求信号
  • Phase 0中对"供给约束"的讨论 — 这是Node 1-6的领域，Node 7应聚焦需求
  • Nadella 49% OpenAI stake — 作为投资关系有趣，但对需求量化没有帮助
#15 该补什么
  1. 中国需求侧简要覆盖（#24 DeepSeek效率突破 + #25 ByteDance规模 + #26 Alibaba云 + #28 Tencent）
  2. LOI vs 合同硬度标注系统（在三角验证表中用颜色区分）
  3. Copilot daily active vs seats数据（如果可获取）
  4. 泡沫场景具体传导链
  5. Musk可信度重新校准（花销 vs 承诺分开评）

最终方法论 — 需求侧5条规则

规则1：买家验证 > 卖家预测
Nadella的审计CapEx比Jensen的$1T订单更可信。但买家也有动机——需要对两方都打折。

规则2：LOI ≠ 合同 ≠ 实际花费
$100B LOI × 60%转化率 = $60B。这仍然巨大，但叙事不同。

规则3：Microsoft CapEx是最快的需求指标
比NVIDIA财报早1-2个月。比TSMC月度数据多1-2个月背景（CapEx指引是前瞻，TSMC是滞后）。

规则4：Copilot续约率是最早的微观崩塌信号
如果企业不续约Copilot → AI应用ROI不成立 → CapEx最终不可持续。

规则5：当所有人都看多时，准备好反向脚本
买家卖家分析师投资者全部同向 = 有史以来最拥挤的多头。保留15-20%的泡沫场景意识。
