§1 节点定义与边界 Node Definition & Boundaries

核心定义 Core Definition：金融科技支付（Fintech Payments）指利用技术创新的数字支付解决方案，涵盖移动钱包、跨境汇款、B2B支付、嵌入式金融及加密支付等细分领域。本节点聚焦于非传统金融机构主导的支付创新，与传统银行支付形成差异化定位。

英文定义 English：Fintech Payments refers to technology-driven digital payment solutions including mobile wallets, cross-border transfers, B2B payments, embedded finance, and crypto payments. This node focuses on non-traditional financial institutions' payment innovations.

边界界定 Boundaries：

  • ✅ 包含 Included：数字钱包、实时支付、QR码支付、BNPL、支付网关、API支付、BaaS
  • ❌ 排除 Excluded：传统银行核心系统（→Banking Node）、纯加密交易所（→Crypto Node）、电商平台内部支付（→E-commerce Node）

时间边界 Temporal Scope：2015年至今（移动支付爆发期），重点关注2019-2026年数据

§3 认知偏差 Cognitive Biases

Critical 严重 地理偏差 Geographic Bias：英语信息源主导导致美国/欧洲案例过度代表，亚太特别是东南亚、非洲、中东的支付创新被系统性低估。Alipay/WeChat Pay的成功往往被简化为"中国故事"，忽视其普惠金融创新价值。

Critical 严重 光环效应偏差 Halo Effect：Stripe、Klarna等明星公司的媒体报道密集，容易高估其在整体市场的影响力。实际上大量区域性支付玩家（如东南亚的OVO、GCash）服务着更庞大的人口基数。

Warning 警告 技术决定论 Tech Determinism：过度关注API、区块链等技术创新，而忽视监管框架、消费者习惯、基础设施等非技术因素对支付格局的决定性影响。

Warning 警告 可获得性启发 Availability Heuristic：近期新闻（如Klarna亏损、Stripe裁员）容易被放大为行业危机信号，而忽视长期趋势和结构性增长。

Warning 警告 幸存者偏差 Survivorship Bias：成功案例（独角兽）被深入研究，失败案例和退出者（如Wirecard、Plaid失败的Stripe收购）分析不足。

Confirmed 已确认 确认偏差 Confirmation：研究前预设"支付去中心化趋势"的叙事框架，可能选择性关注支持性证据而忽视反例。

§4 上下游交叉 Upstream/Downstream Cross-References

→ 监管节点 RegTech Node
支付合规（PCI-DSS、KYC/AML）与RegTech紧密关联。欧盟PSD2、美国CFPB监管推动开放式银行（Open Banking）发展，直接影响支付创新路径。
→ 银行即服务 Banking-as-a-Service
BaaS（Synapse、Unit等）将支付基础设施模块化，使嵌入式支付成为可能。BaaS与Fintech Payments存在大量重叠。
→ 电商节点 E-commerce Node
电商平台是支付的重要应用场景。Shopify Payments、Shopify Checkout反映电商与支付深度整合趋势。
→ 加密节点 Crypto Node
稳定币（USDC、USDT）、CBDC与跨境支付存在交集。Ripple（XRP）、Stellar定位为跨境支付解决方案，与传统Fintech支付竞争。
→ 普惠金融节点 Financial Inclusion
非洲M-Pesa、南亚UPI等案例体现支付作为金融普惠基础设施的角色，与Fintech Payments互为补充。

交叉验证：与Blockchain Hub中Crypto Payment节点进行数据互校，重点对齐跨境支付市场规模估算。

§7 决策日志 Decision Log

2026-03-15 | 节点边界决策 Node Scope Decision
决定将"嵌入式支付（Embedded Payments）"纳入本节点而非单独成节点。理由：嵌入式金融边界模糊，且支付是其最成熟应用场景。风险：可能稀释B2B支付的核心地位。
2026-03-16 | 公司归类决策 Company Classification Decision
采用"业务模式×地域×客群"三维矩阵代替简单的市值排名。触发点：Stripe（高估值但B2B集中）与Mercado Pago（地域性强但生态完整）难以用单一维度比较。效果：分类更具分析价值，但增加读者认知负担。
2026-03-17 | BNPL争议处理 BNPL Controversy Handling
决定给予BNPL独立子章节，因其商业模式争议性（消费者债务、监管套利）需要专项讨论。替代方案（并入支付方式）被否决：BNPL已成为独立赛道，与传统支付有本质差异。
2026-03-18 | 加密支付边界 Crypto Payments Boundary
